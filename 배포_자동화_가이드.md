- ✅ GitHub 저장소: https://github.com/neungju241601-afk/washing
- ✅ Render 계정 (무료로 만들 수 있어요!)

## 🎯 Render에 배포하기 (가장 쉬운 방법!)

### 1단계: Render 계정 만들기

1. **Render 웹사이트 접속**
   - https://render.com 접속
   - "Get Started for Free" 클릭

2. **GitHub로 로그인**
   - "Continue with GitHub" 클릭
   - GitHub 계정으로 로그인
   - Render가 GitHub 저장소에 접근할 수 있도록 권한 허용

---

### 2단계: 새 웹 서비스 만들기

1. **대시보드에서 "New +" 버튼 클릭**
   - 화면 오른쪽 위 또는 왼쪽 사이드바

2. **"Web Service" 선택**

3. **GitHub 저장소 연결**
   - "Connect account" 클릭 (처음만)
   - "Connect repository" 클릭
   - `neungju241601-afk/washing` 저장소 찾기
   - "Connect" 클릭

---

### 3단계: 서비스 설정

다음 정보를 입력하세요:

#### 기본 설정
- **Name**: `washing-machine` (원하는 이름으로 변경 가능)
- **Region**: `Singapore` (한국과 가장 가까운 지역)
- **Branch**: `main` (기본값)
- **Root Directory**: (비워두기)

#### 빌드 및 시작 명령어
- **Runtime**: `Python 3`
- **Build Command**: 
  ```
  pip install -r requirements.txt
  ```
- **Start Command**: 
  ```
  python app.py
  ```

#### 환경 변수 (Advanced 섹션)
"Advanced" 버튼을 클릭하고 다음 환경 변수 추가:

| Key | Value |
|-----|-------|
| `FLASK_DEBUG` | `False` |
| `PORT` | (비워두기 - Render가 자동 설정) |

---

### 4단계: 배포 시작!

1. **"Create Web Service" 버튼 클릭**
   - 배포가 자동으로 시작됩니다!

2. **배포 진행 상황 확인**
   - 로그를 보면서 배포 진행 상황 확인
   - 약 2-3분 정도 소요됩니다

3. **배포 완료!** 🎉
   - "Your service is live!" 메시지가 나오면 성공!
   - 주소가 생성됩니다: `https://washing-machine.onrender.com` (또는 설정한 이름)

---

## 🌐 배포된 주소 사용하기

### 주소 형식
```
https://[서비스이름].onrender.com
```

예시:

### 접속 확인
1. 생성된 주소를 브라우저에서 열기
2. 세탁기 예약 시스템 화면이 보이면 성공!

## ⚙️ 추가 설정 (선택사항)

### 커스텀 도메인 연결
1. Render 대시보드에서 서비스 선택
2. "Settings" 탭 클릭
3. "Custom Domain" 섹션에서 도메인 추가
   - ⚠️ 도메인은 별도로 구매해야 합니다

### 자동 배포 설정
- 기본적으로 GitHub에 push하면 자동으로 재배포됩니다!
- "Settings" → "Auto-Deploy" 확인

## 🔄 업데이트 배포하기

코드를 수정한 후:

1. **로컬에서 수정**
   ```bash
   # 파일 수정 후
   git add .
   git commit -m "수정 내용"
   git push
   ```

2. **자동 재배포**

   - 자동으로 재배포 시작!

## 🆘 문제 해결

### "Build failed" 오류
- **원인**: 패키지 설치 실패
- **해결**: `requirements.txt` 확인, 로그 확인

### "Application error" 오류
- **원인**: 앱 실행 실패
- **해결**: 
  1. "Logs" 탭에서 오류 메시지 확인
  2. `app.py`의 포트 설정 확인
  3. 환경 변수 확인

### 서비스가 느려요
- **원인**: 무료 플랜은 15분 비활성 시 슬리프 모드
- **해결**: 
  - 첫 접속 시 약간의 시간이 걸릴 수 있어요 (깨어나는 시간)
  - 유료 플랜으로 업그레이드하면 항상 켜져있어요

---

## 📊 Render 무료 플랜 제한사항

| 항목 | 제한 |
|------|------|
| **서비스 수** | 무제한 |
| **비활성 시간** | 15분 후 슬리프 모드 |
| **트래픽** | 무제한 |
| **빌드 시간** | 750시간/월 |
| **디스크** | 512MB |

**참고**: 슬리프 모드는 첫 접속 시 깨어나는 데 약간의 시간이 걸릴 수 있어요.
